# Contributing
Thanks for your interest in contributing.

## Steps
1. Fork the repo and create a new branch (preferably linking to the issue):
   ```bash
   git checkout -b 42-your-feature
   ```
2. Make your changes and test locally.
3. Follow commit style:
   * feat: add feature
   * fix: resolve bug
   * docs: update docs
4. Push and open a Pull Request to main.
5. Link related issues (e.g., Closes #42).

## Guidelines
* Keep PRs small and focused.
* Match the existing code style.
* Update docs or tests if affected.
